Problem Solving

This blog post will be focused on problem solving. It will deal with different techniques as well as personal experiences of their applications.

Problem Solving Techniques:


1: Pseudocode: Pseudocode is not actual code, but rather comments that tell you how the code will eventually look. It is informal, plainly written steps that form the rough draft or outline of your code.

2: The rubber ducky method: This one is talking thorugh or explaining your code/process to an inanimate object or person without any experience in what your doing in a way that allows them to understand. this method works on the idea that when you can explain something so well it sounds easy then do you truly understand what you are doing.

3: Reading error messages: This one is self-explanatory

4: Console.logging: The console is the built-in debugger of the browser. Many developers use console.log() all the time to print messages and debug problems in their code.

5: Googling: we all know this one

6: Trying something: sometimes you just gotta play with the code and see what happens.

7/8: Asking your peers for help/Asking your coaches or managers or whatever for help

9: Improving your process with reflections: which means to look back at how you created your code and just trying to understand your choices around it.

Tell your non-tech friend a story about a time you:

1: Got blocked on a simple problem:

a time i got blocked and sometimes still happens is when my css code stopped working. at the time the links to my style sheets would not connect with the html, so no matter what i did my css just would not express on the webpage.

*what problem solving techniques did you use?

the main techniques i used was googling, asking for help and reading error messages.
reading the error messages was i learnt that it was the actual link that was broken but the code was correct so i tried googling my problem on different websites but nothing seemed to work or help, i then asked my brother who is a software analist what i should try and he was unable to come up with anything. it wasn't until i used 'try something' that i was finally able to fix it by just deleting the code attached to css and rewriting it.

*How did you feel throughout the process?

it was frustrating as it was putting my over time and it just felt like it would not work but after finally just scrapping the link and rewriting it, it felt good to finally seeing it work.

*What did you learn?

i learnt to pace myself better when dealing with something that isn't working as i can often find myself often just trying to force myself to work through it, and that sometimes its okay to just try something different with the code since its already broken.
